1. Overview

CleanAdsSurf is designed as a privacy-first Chrome extension. Core filtering, domain controls, and local stats run in your browser environment.

2. Data Collected

By default, CleanAdsSurf does not sell personal browsing data and does not require remote account usage for core blocking features.

If optional authentication demo pages are used, submitted credentials are handled by your configured backend service.

3. Local Storage

The extension stores operational values in chrome.storage.local, such as:

  • Global protection setting
  • Per-domain override values
  • Theme preference
  • Daily and total blocked counters

4. Permissions Used

  • storage: save local settings and counters
  • declarativeNetRequest: apply filtering rules
  • declarativeNetRequestFeedback: rule-match diagnostics
  • tabs: detect active tab domain for toggles
  • <all_urls>: evaluate requests against filtering rules

5. Third-Party Requests

Optional utility pages (for example speed/network tools) may request public endpoints to provide those features. These calls are feature-driven and not hidden analytics.

6. Retention

Local extension data remains on your browser profile until you clear extension data or uninstall the extension.

7. Policy Changes

This policy can be updated when features or architecture changes. The date above reflects the latest revision.

8. Contact

For privacy questions, use the repository issue tracker: CleanAdsSurf on GitHub.
